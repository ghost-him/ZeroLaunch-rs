# 前端国际化强制规范

所有用户可见的文本字符串 **必须** 通过 `t('key')` / `$t('key')` 或 `{{ $t('key') }}` 渲染。**禁止** 在模板或脚本中硬编码面向用户的文字。

## 适用范围

| 内容 | 必须 i18n | 允许硬编码 |
|------|-----------|-----------|
| 按钮标签、标题、描述、占位符 | ✓ | — |
| 提示信息、错误消息、通知文本 | ✓ | — |
| 表格列标题、表单字段标签 | ✓ | — |
| 工具提示、空状态说明 | ✓ | — |
| 日志、调试输出、内部错误码 | — | ✓ |
| HTML `alt` 属性、`aria-label` 等可访问性标签 | — | ✓（若纯标识性） |
| 语言文件 key 名、组件 prop key | — | ✓ |
| CSS `content` 属性中的文字 | ✓ | — |

## 判断标准：什么算"用户可见"

如果一个字符串最终出现在浏览器的渲染树中（包括通过 `v-if`、`:title`、`:placeholder` 等间接渲染），它就是用户可见的，**必须** 走 i18n。

以下 **不** 算用户可见，可硬编码：
- 纯标识性的 key、id、类名
- 日志消息
- 后端传来的数据（后端负责其内容的国际化）

## 工作流

### 1. 在语言文件中添加条目

```typescript
// src-ui/i18n/locales/zh-Hans.ts
export default {
  alias: {
    title: '别名配置',
    searchPlaceholder: '搜索程序...',
    noResults: '未找到匹配的程序',
  },
}
```

```typescript
// src-ui/i18n/locales/en.ts
export default {
  alias: {
    title: 'Alias Configuration',
    searchPlaceholder: 'Search programs...',
    noResults: 'No matching programs found',
  },
}
```

### 2. 在模板或脚本中引用

```vue
<!-- 模板中 -->
<n-button>{{ $t('alias.title') }}</n-button>
<n-input :placeholder="$t('alias.searchPlaceholder')" />

<!-- script setup 中 -->
<script setup lang="ts">
import { useI18n } from 'vue-i18n'
const { t } = useI18n()
const placeholder = t('alias.searchPlaceholder')
</script>
```

### 3. 新增文本必须同步添加两种语言

**禁止** 只加中文不加入英文。两种语言文件必须同步更新，同一 commit 中完成。

## 错误示例（禁止）

```vue
<!-- ❌ 硬编码中文 -->
<n-empty description="未找到匹配的程序" />

<!-- ❌ 硬编码任何语言 -->
<n-button>添加</n-button>
<n-input placeholder="搜索..." />

<!-- ❌ 模板字符串拼接 -->
<div>{{ '共 ' + count + ' 个结果' }}</div>
```

## 正确示例

```vue
<!-- ✓ 通过 i18n key -->
<n-empty :description="$t('common.noResults')" />
<n-button>{{ $t('common.add') }}</n-button>
<n-input :placeholder="$t('alias.searchPlaceholder')" />

<!-- ✓ 带参数的翻译 -->
<div>{{ $t('common.resultCount', { count }) }}</div>
```

## 语言文件结构约定

- 按功能域或组件名组织顶层 key，避免单一扁平命名空间
- key 使用 camelCase
- 通用文本放 `common` 命名空间：`common.save`、`common.cancel`、`common.delete`
- 组件/页面专属文本放组件名命名空间：`alias.title`、`alias.searchPlaceholder`

## 审查清单

新增或修改 Vue/TS 文件后自查：
- [ ] 所有模板字符串（`{{ }}`）中的用户可见文本是否走 `$t()`？
- [ ] 所有 `:placeholder`、`:title`、`:description`、`:label` 等绑定是否走 `t()`？
- [ ] `n-empty`、`n-text` 等组件的 `description`、label 属性是否走 i18n？
- [ ] 脚本中的警告/通知字符串是否走 `t()`？
- [ ] 两种语言文件是否都已同步更新？
